Claims Meeting Summariser

Mind Map & Process Map — Project Planning

Mind Map
Process Map
Phase 1 — User Input
01
Open app
User opens the React SPA in browser. No login required for demo.
›
02
Select meeting type
Claims review, executive briefing, or team standup. Sets the output template and prompt context.
›
03
Paste transcript
User pastes raw meeting text into the input field. Plain text, no formatting required.
›
04
Add context (optional)
Meeting date, attendees, claim reference numbers. Improves output specificity.
Phase 2 — Processing
05
Build prompt
App constructs a structured prompt: system role (Claims analyst), meeting type context, user transcript, and output schema.
›
06
Call Claude API
POST to Anthropic Messages API (claude-sonnet-4-20250514). Instructs model to return structured JSON.
›
07
Parse response
App extracts JSON fields: summary, decisions, action items, owners, follow-up dates. Handles malformed output gracefully.
Phase 3 — Decision Gate
Did API return valid structured output?
Yes → Render summary view
No → Show error, allow retry with revised input
Phase 4 — Output
08
Summary block
2–3 sentence plain-language summary of what was discussed and decided.
›
09
Decisions table
Bulleted list of decisions made. Each flagged as confirmed, pending, or deferred.
›
10
Action items
Each item: task description, owner name, due date. Formatted as a table.
›
11
Export
Copy to clipboard as Markdown or download as .txt. Future: push to Teams channel.
Phase 5 — Feedback Loop
12
User reviews output
User reads summary. If inaccurate, they can edit inline or flag specific sections.
›
13
Regenerate (optional)
User adjusts context or meeting type and resubmits. API called again with refined prompt.
›
14
Save / share
Final output copied or downloaded. Session ends. No data stored server-side.
