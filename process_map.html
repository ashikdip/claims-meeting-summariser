Process Map · v1.3
Process Map
How it works
End-to-end flow from transcript input to structured output and export — Claims Meeting Summariser v1.3
1
Phase 1
Setup
Step 01
Open the app
User opens the standalone HTML app via GitHub Pages. No installation, no login, no account required.
User action
Step 02
Paste API key
User pastes either an Anthropic key (sk-ant-...) or a Google Gemini key. The app detects the provider automatically and shows a badge.
Auto-detected
2
Phase 2
Input
Step 03
Select meeting type
User chooses Claims Review, Executive Briefing, or Team Standup. Each type loads a different system prompt context.
User action
Step 04
Paste transcript
User pastes plain meeting transcript text. Live word count displayed. No special formatting needed.
User action
Step 05
Submit
User clicks Summarise Meeting. Button enters loading state with spinner. Input is locked during processing.
User action
3
Phase 3
Processing
Step 06
Build prompt
App combines the meeting type system context with the transcript and a strict JSON output schema including summary, decisions, action items, and follow-up dates.
Automated
Step 07
Route to API
Based on key prefix, request is sent to Anthropic Messages API (Claude Sonnet 4) or Google Gemini API (Gemini 2.5 Flash) with JSON response mode enforced.
API call
Step 08
Parse response
Raw JSON string is stripped of markdown fences and parsed. Fields are extracted: summary, decisions[], action_items[], follow_up_dates[].
Automated
⚡
Decision gate — Did the API return valid structured output?
Yes
Structured output rendered in the UI. User proceeds to review and export.
No
Error message shown with the API error detail. User can adjust transcript or retry.
4
Phase 4
Output
Step 09
Summary
4-5 sentence summary covering main topics, key context, and overall outcome. Copy button on the section header.
Rendered
Step 10
Decisions
Each decision tagged with a coloured status badge — Confirmed (blue), Pending (red), Deferred (grey). Item count shown in header.
Rendered
Step 11
Action items
Table with three columns: Task, Owner (blue pill), Due Date. Alternating row shading. Item count shown in header.
Rendered
Step 12
Follow-up dates
Each date rendered as a chip with context label in parentheses — e.g. May 20, 2026 (Q2 prep session).
Rendered
5
Phase 5
Export
📋
Copy Text
Full summary copied to clipboard as plain text. All four sections included, labelled and separated.
Clipboard API
📄
Export DOCX
Formatted Word document with HP-styled section headers, coloured decision badges, and action items table. Downloaded directly.
docx.js v7
🖨️
Export PDF
A4 PDF with Electric Blue header band, double-slash motif, coloured badges, vertically centred table rows, follow-up date chips, and page footer.
jsPDF v2.5
